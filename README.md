From SQL Interrogation to Machine Learning: Cracking the Customer Churn Code 🔍
I recently completed a deep-dive analytics project to solve one of the most expensive problems in Telecom: Subscriber Churn.
Rather than just "running a report," I built a full-stack predictive pipeline to move from raw data to actionable CEO-level strategy.
The Tech Stack:

Database: SQL (Standard DML)
IDE: [SQLiteOnline]
AI Engine: Python (Pandas & Scikit-learn)
My Process:
1️⃣ SQL Feature Selection: I performed "Noise Cancellation" by proving that features like gender and multiple phone lines were statistically insignificant.
2️⃣ Signal Discovery: Identified "High-Risk" behaviors. For example, Fiber Optic users without Online Security churn at a massive 41% rate compared to just 14% for those with security "anchors."
3️⃣ Predictive Modelling: Exported a high-signal dataset into a Random Forest model, achieving a 77.3% accuracy rate in predicting customer exits.
The "So What?" (Business Impact):
The AI confirmed my SQL findings: Contract Type and Value-Added Services (Tech Support/Security) are the primary drivers of loyalty. My recommendation? Bundle security features with high-speed plans to create "product stickiness" and bridge the 17-month "danger zone."
